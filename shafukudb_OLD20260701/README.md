# shafukudb — 照合層(フェーズ2)

社会福祉法人の財務PDF抽出行を、正典マスタ `seiten_master.csv`(691行)の
正典コードへ対応づけるクリーンな照合層。旧 `deliver/` の失敗作とは設計を
共有しない、ゼロベース再構築。

## モジュール
- `normalize.py` — 科目名の正規化。**正典側・実PDF側の双方で必ず通す**。
  意味括弧(公費/一般/保険等査定減)は保持し、算式番号・恒等式・連番のみ除去。
  中黒「・」は科目名の一部として保持(旧 naming.py の無条件削除は不採用)。
- `seiten.py` — 正典ローダ。2本のインデックスを構築:
  「計算書×正規化名→候補(1対多)」と「親パス→子レコード」。
- `matcher.py` — 照合エンジン。優先順位: 親スコープ内一致 → 計算書×名の候補 →
  複数残れば曖昧退避。未知行退避理由は NO_NAME / AMBIGUOUS / STRUCTURE / MISEXTRACT。
  様式別の期待最深階層 `FORM_MAX_DEPTH` を保持(華野で実証の5様式を起点)。
- `test_self.py` — 正典自己テスト。抽出品質と独立に照合の正しさを検証。

## 検証結果(フェーズ2-1)
- (A) 親パスあり: 全691行が元codeへ厳密復元。誤code 0・未ヒット 0。
- (B) 親パスなし: 197行が曖昧化 → 親パス照合が必須であることを定量実証。
- 正規化は正典内で NO_NAME=0(消しすぎ・残しすぎなし)。

## 責務の外
PDF抽出(罫線・x座標・インデント深度)と算式検算(恒等式)は照合層の外。
次段階で華野福祉会の実PDF抽出に接続する。

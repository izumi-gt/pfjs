# マスタ コード体系フォーマット定義書（masters.py 拡充用）

このドキュメントは、社会福祉法人 決算DB化プロジェクトの標準勘定科目マスタ
（`shafuku_db_engine/masters.py`）を、**社会福祉法人会計基準の別表（正典）に沿って
体系的に拡充する**ための、コード採番ルールと記述フォーマットの定義です。

新チャットはこの定義に厳密に従ってマスタ行を生成すること。**自由なコード採番をしない。**

---

## 0. 大前提（絶対に守る）

- マスタは「**幹（global stem）= 法人ごとに揺れない大区分/中区分/計/差額/残高**」のみを保持する。
  - CF/PL: **大区分**のみ（小区分・明細は法人別 local として別管理。マスタに入れない）。
  - BS: **中区分**まで（小区分は入れない）。
- 勘定科目は会計基準別表で**固定されたクローズドな集合**。別表にある正規科目だけを追加する。
  法人独自の造語・表記ゆれは追加しない（それらは local/review_queue 側で吸収される）。
- **既存行は一切変更・削除しない。末尾に追記するだけ**（採番順・出力順を変えると
  神奈川基準線の sha256 が崩れるため）。既存の連番の隙間を埋めるのも避け、原則新規枝番で足す。

## 1. テーブル構造（タプルの並び）

### CF_MASTER（資金収支計算書）
```python
(code, 活動区分, 収支区分, 正規名, is_total)
```
- `活動区分`: `'A'`(事業活動) / `'B'`(施設整備等) / `'C'`(その他の活動) / `'X'`(合計・残高)
- `収支区分`: `'収入'` / `'支出'` / `'差額'` / `'残高'`
- `正規名`: 別表の科目名（中黒「・」はそのまま書いてよい。normalizeで除去され突合される）
- `is_total`: 計・差額・残高など集計行なら `1`、通常の大区分なら `0`

### PL_MASTER（事業活動計算書）
```python
(code, 活動区分, 収益費用区分, 正規名, is_total)
```
- `活動区分`: `'S'`(サービス活動増減) / `'SO'`(サービス活動外増減) / `'T'`(特別増減) /
  `'K'`(繰越活動増減差額) / `'KEI'`(経常増減差額)
- `収益費用区分`: `'収益'` / `'費用'` / `'差額'`
- `正規名` / `is_total`: CFと同様

### BS_MASTER（貸借対照表）
```python
(code, 区分, 中区分, 正規名, is_total, indent)
```
- `区分`: `'ASSET'`(資産) / `'LIAB'`(負債) / `'NET'`(純資産)
- `中区分`: 例 `'流動資産'` `'固定資産'` `'基本財産'` `'その他の固定資産'` `'流動負債'` 等
- `is_total`: 小計・合計なら `1`、通常項目なら `0`
- `indent`: 階層インデント（0=大区分見出し, 1=中区分, 2=小区分相当）。既存の同区分行に合わせる。

## 2. コード採番ルール

### 共通形
```
CF: CF-{活動区分}-{収支3文字}-{連番3桁}        例 CF-A-INC-140
PL: PL-{活動区分}-{収益費用3文字}-{連番3桁}     例 PL-S-REV-110
BS: BS-{区分1文字}-{中区分略号}-{連番3桁}       例 BS-A-CUR-160
```

### 収支区分の3文字略号
| 計算書 | 区分 | 略号 |
|---|---|---|
| CF | 収入 | `INC` |
| CF | 支出 | `EXP` |
| PL | 収益 | `REV` |
| PL | 費用 | `EXP` |

### 計・差額・残高の特殊サフィックス（既存踏襲）
- `-T` = 計（例 `CF-A-INC-T` 事業活動収入計）
- `-NET` = 差額（例 `CF-A-NET`）
- `CF-X-TOT/BEG/END` = 当期資金収支差額合計/前期末/当期末支払資金残高
- これらは既存に揃っているので**新規追加は通常不要**。

### 連番の振り方（最重要）
- **既存の最大連番の続きから振る**。既存が 010,020,...,130 まで使い、追加で末尾に
  足した枝番（005, 015, 020b 等）もあるため、**衝突しないことを必ず確認**してから採番する。
- 別表の科目順を尊重しつつ、コードは「既存の続き番号」で良い（コードの数値順と
  別表の並び順が完全一致している必要はない。突合は正規名で行うため）。
- 既存にある枝番方式（`CF-A-INC-005`, `CF-B-INC-020b` 等）に倣い、
  どうしても間に入れたい場合のみ枝番（例 `015`, `020b`）を使う。原則は末尾連番。

## 3. 正規名の書き方（突合の要）

`resolve_global(name)` は `normalize(name)` の結果でマスタと突合する。
新マスタの正規名も、**normalize後に取り込み側の正規化キーと一致**しなければ効かない。

normalizeの主な作用（参考）:
- NFKC正規化（全角英数記号→半角）、全角/半角空白・タブ除去
- 末尾の算式注記除去（`＝（３）＋（６）` 等、最初の `=`/`＝` 以降を削除）
- 連番括弧除去（`（１）` `(1)` `（一）` 等）、空括弧除去
- 中黒「・」除去、ハイフン類統一、旧字統一（齋→斎 等）

実務上の指針:
- **別表の科目名をそのまま正規名に書けばよい**（中黒や括弧があっても normalize が吸収）。
  例: 別表「介護予防・日常生活支援総合事業収入」→ 正規名にそのまま記載。
  集計キーは「介護予防日常生活支援総合事業収入」（中黒除去後）で、突合は成立する。
- 末尾に `（１）` のような連番が付く計・差額行は、連番を**付けずに**書く
  （normalizeで落ちるため。既存の計・差額行も連番なしで書かれている）。

## 4. 作るもの（成果物）

新チャットは以下を生成する:

1. **CF_MASTER / PL_MASTER / BS_MASTER への追加タプル列**（Pythonリテラルで、
   そのまま既存リストの末尾に貼れる形）。各タプルに `# 法人数NNN（440社集計より）`
   のコメントを付け、根拠を残す。
2. 追加した大区分の **正規名 ↔ 採番コード の対応表**（後で検証に使う）。
3. 別表に存在するが440社集計に出なかった大区分があれば、**追加するかの判断materを併記**
   （2万社展開を見据え、別表完全網羅を優先するか、実出現のみに絞るかの論点）。

**コードは書き換えず、追加タプルだけを出力する。** 実際の masters.py への反映と
神奈川基準線の再確認は、結果を受け取った後にこちら（元チャット）側で行う。

---

## 5. 検証観点（新チャットが自己チェックすべき点）

- 追加コードが既存コードと**重複していないか**（CF/PL/BS各リストで全code集合を取り重複0を確認）。
- 正規名が既存マスタの正規名と**normalize後に衝突していないか**
  （衝突すると resolve_global が既存に吸われ、新規が死ぬ or 取り違える）。
- 大区分のみを足しているか（小区分・明細を混入させていないか）。
- タプルの要素数・型が既存と一致しているか（CF/PL=5要素, BS=6要素）。

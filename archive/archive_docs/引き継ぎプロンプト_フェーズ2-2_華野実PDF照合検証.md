# 引き継ぎプロンプト（フェーズ2-2：華野福祉会 実PDFでの照合検証）

> 再構築ロードマップ フェーズ2「照合パーサへの作り替え」の第2段階。
> 前段階フェーズ2-1で照合層(`shafukudb/`)をゼロベース実装し、正典自己テストで
> 「親パスありで全691行が元codeへ厳密復元」を確認済み。本段階はそれを**実PDFに接続**する。

このチャットは、照合層を実PDFで初めて走らせる段階です。対象は介護保険系の
**社会福祉法人華野福祉会(広島県・2拠点)**の12様式。まずプロジェクト内の
`seiten_master.csv`、`フェーズ2-1_照合層のゼロベース実装.md` と、GitHub
`izumi-gt/pfjs` の `shafukudb/` を読み、下記の前提と整合するか確認してから着手すること。

---

## 0. 前段階で確定していること(踏襲する)

- 照合層は `izumi-gt/pfjs` の **`shafukudb/`**（`deliver/` は失敗作の参照用に残置）。
  - `normalize.py`(正典側・実PDF側の双方で必ず通す) / `seiten.py`(正典ローダ＋
    by_norm:1対多 ＋ by_parent) / `matcher.py`(照合エンジン＋退避4区分＋FORM_MAX_DEPTH) /
    `test_self.py`(正典自己テスト)。
- 照合の優先順位：親スコープ内一致 → 計算書×正規化名の候補 → 複数残れば AMBIGUOUS 退避。
- 退避4区分：NO_NAME / AMBIGUOUS / STRUCTURE / MISEXTRACT。
- 正規化：意味括弧(公費/一般/保険等査定減)保持、中黒「・」保持、連番・算式のみ除去。
- is_total は照合対象(算式検算は別レイヤ)。is_optional は欠落扱いしない。
- 正典自己テスト結果：親パスあり=全691行厳密復元(誤0/未0)、親パスなし=197行曖昧化。

## 1. このチャットのスコープ(フェーズ2-2)

照合層の**外側の部品**＝「実PDF → 行＋インデント深度 → 親スコープ復元」を実装し、
華野の12様式を照合層に通して結果を検証する。

- 実PDF抽出：罫線・x座標からインデント深度を検出し(旧 `hierarchy.build_indent_levels`
  /`level_of` は部品として採否判断)、各行の親スコープ(計算書,L0..L3.5 のコード列)を
  復元する。**親スコープの復元品質が照合の成否を決める**。
- 様式別期待深度：華野で実証済みの5様式(1-1/1-3=大区分, 1-4=小区分, 3-1〜3-3=中区分,
  3-4=小区分)を起点に、残り様式(1-2, 2-1〜2-4 等)を**実物で確認しながら** FORM_MAX_DEPTH
  に追加(推測で埋めない)。

明示的に対象外：算式検算(恒等式)、主キー設計の最終確定、大量取込。

## 2. 検証で確認すべきこと(前段階からの申し送り)

- 華野12様式の各行が、親スコープ経由で正典codeへ正しく引けること。
- `その他の事業収入`(正典で9箇所同名)等の同名重複が、親パスで一意化されること。
- BS同名重複(貸倒引当金 等)が多重マッチせず一意化されること(前回の主キー潰れ再現テスト)。
- `予備費支出`(1-1のみ)が is_optional として欠落扱いされないこと。
- CF/PL の科目名差(退職給付支出 ↔ 退職給付費用＋非資金)が計算書別に正しく引けること。
- 未知行が4区分のどれで退避されるか、その内訳をレビューする。

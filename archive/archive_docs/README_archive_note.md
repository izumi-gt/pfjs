# archive/ 配置ノート

このフォルダの6文書は、社福決算DB化プロジェクトのプロジェクトナレッジから**外した**もの。
削除はしていない（`deliver/`を失敗作の参照用として残す既存方針と同じ扱い）。

**ナレッジから外した理由**：Claude(このプロジェクトのAI)には
「プロジェクトナレッジを最優先・最も権威ある情報源として検索する」という仕組みがある。
古い設計・放棄した手法がナレッジに残ると、次のチャットが検索で拾い、
**否定済みの前提を正しい前提として採用してしまう危険が現実にある**（特に3・6は要注意）。
歴史的経緯・失敗の記録としての価値は残るため、削除せずここに退避した。

置き場所：`izumi-gt/pfjs` リポジトリの `archive/` 配下（`deliver/` と並置）に置くことを想定。

## 各文書の状態

### 1. マスタ拡充_フォーマット定義書.md — 完全に古い
旧`masters.py`のコード体系（`CF-A-INC-140`形式、5要素タプル）の採番ルール書。
現行の正典マスタ `seiten_master_v3.csv` は全く別の6セグメントcode
（`CF-01-01-001-000-000`形式、L0=計算書プレフィックス）であり、本文書のルールは適用できない。

### 2. フェーズ1_正典マスタ作成.md — 完全に古い・要注意
e-Gov別表由来の旧691行マスタ(`seiten_master.csv`、19列)の構築記録。
**本文書のL0=収入/支出、L1=活動・部という定義は、現行v3マスタのL0=CF/PL/BS、
L1=活動・部という定義と異なる。同じ用語（L0, L1）が別のものを指す点に注意。**
現行の唯一の正本は `seiten_master_v3.csv`（WAM「財務諸表等入力シート」由来）。

### 3. フェーズ1_5_集計行の正典反映.md — 完全に古い
旧691行マスタへのis_total/is_optional付与記録。旧コード体系（`BS-08-00-04-01-00-00`等）。
現行v3マスタの集計行判定はWAMのExcel数式(N列)が一次情報で、`合算定義`列に
符号付き子codeのJSON配列として反映済み（本文書の手法とは別物）。

### 4. フェーズ2-1_照合層のゼロベース実装.md — 完全に古い・要注意
旧`shafukudb/`（`normalize.py`/`seiten.py`/`matcher.py`/691行マスタ）の設計記録。
**「インデント深度から親スコープを復元する」方式（`FORM_MAX_DEPTH`等）を採用しているが、
これは様式1-4読み取り機確立の過程で「WAM生成PDFでは原理的に読めない」と判明し、
放棄した手法である。** 現行は「横書き科目名のみで2ポインタ照合」方式（`matching/`）。

### 5. 引き継ぎプロンプト_フェーズ2-2_華野実PDF照合検証.md — 完全に古い・最も要注意
次チャットに「実PDF→インデント深度→親スコープ復元」の実装を指示する引き継ぎ文書。
**これは様式1-4読み取り機確立のセッションで実際に何時間もかけて試行し、
構造的に破綻することが判明して放棄した道そのものである。** 次のチャットがこれを読んで
同じ失敗を再現するリスクが最も高い文書。

### 6. 神奈川基準線_検証コード.md — 完全に古い（実体消失）
さらに前の実装`shafuku_db_engine`（`deliver/`より前）への回帰チェックコード。
参照先のモジュール自体が現行構成に存在しない。歴史的記録としてのみ保存。

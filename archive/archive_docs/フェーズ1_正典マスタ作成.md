# フェーズ1：正典マスタ作成（確定記録）

このドキュメントは、社会福祉法人 決算DB化プロジェクト フェーズ1において、
e-Gov 取得の法令データ（`seiten_kamoku.csv`）から**正典マスタ**を作成した際の
検討内容・確定事項・成果物を記録したものです。後続チャットはまずこれを読むこと。

- **このチャットのゴール**：正典マスタを完成させる。
- **やらないこと**：パーサ照合・主キー設計・PDF大量取り込み（後続フェーズ）。
- **成果物**：`seiten_master.csv`（正典マスタ本体・660行・19列）。

---

## 1. 入力データ `seiten_kamoku.csv` の把握

e-Gov の社会福祉法人会計基準 別表第一〜第三を構造化した**正典の一次資料**。計660行。
前回失敗の核心（正典の後付け）を解消する、フェーズ1の背骨。

| 計算書 | 別表 | 行数 | テーブル構成 |
|---|---|---|---|
| CF（資金収支計算書） | 別表第一 | 278 | 収入178 / 支出100 |
| PL（事業活動計算書） | 別表第二 | 275 | 収益170 / 費用99 / 繰越等6 |
| BS（貸借対照表） | 別表第三 | 107 | 資産63 / 負債39 / 純資産5 |

元CSVの列：`計算書 / 別表 / テーブル / セクション / 大区分 / 中区分 / 内訳グループ / 小区分 / 階層種別`

**自己整合性の検証結果**（いずれもクリア）：
- 階層種別と列充足の崩れ：**0件**。
- 同一親パス内の小区分重複：**0件**。
- `(計算書,テーブル,大,中,内訳G,小,階層種別)` の7要素で **660行が完全ユニーク**。

**重要な発見（後段への申し送り）**：
- 小区分名は**単独では一意にならない**。`その他の事業収入`等32種が複数大区分に同名出現（最大9箇所）。
- 中区分名も大区分をまたいで同名重複（CF/PLで各8種、`利用者負担金収入`等）。
- → コードは**全レベルで親パス由来**にする必要がある。これが前回の「parent空→
  コード潰れ→主キー重複」の構造的原因であり、ここで断つ。

## 2. 論点の検討と確定事項

### 2-1. 粒度：小区分まで完全構造化（確定）
正典は小区分まで全粒度で持つ。土台の網羅性を高め、法人ごとに揺れる`（何）`系統に
振り回されないようにする。前回の「幹だけ（フォーマット定義書）」方針は教訓資料として
参照に留め、構造・採番はゼロベースで組み直す。

### 2-2. 内訳グループ：独立行として保持（確定）
- 別表にある区分は全部対等に1行持つ。
- 理由：①抽出・集計の軸として使える（内訳のみ抽出、集計からの差し引き）。
  ②金額が来る内訳グループ（`（保険等査定減）`等）の取りこぼしを構造的に防ぎ、
  決算資料と正典が1対1で照合できる状態を作る。
- 副次決定：見出し行と金額行が混在するため、各行に「金額あり/見出し」フラグを付ける
  （後述 2-5）。

### 2-3. 計算書横断の論理階層 L0〜L4（確定）

| レベル | CF | PL | BS |
|---|---|---|---|
| L0 区分 | 収入 / 支出 | 収益 / 費用 / 繰越 | 資産 / 負債 / 純資産（部） |
| L1 活動・部 | 事業活動 / 施設整備等 / その他 | サービス活動 / 同外 / 特別 | （無し） |
| L2 大区分 | 介護保険事業収入 等 | 同 | 流動資産 / 固定資産 等 |
| L3 中区分 | 居宅介護料収入 等 | 同 | 個別科目 |
| L3.5 内訳グループ | （介護報酬収入）等 | 同 | （固定資産のみ・後述） |
| L4 小区分 | 介護報酬収入 等 | 同 | （固定資産配下のみ） |

元CSVの「セクション」列は L0＋L1 に分解して扱う。

### 2-4. BSの「部」と固定資産の階層（実物3社で検証・確定）
- BSの `資産/負債/純資産の部` も**独立行**で持つ（`資産の部合計`等が金額を持つため）。
- **BS固定資産だけが中間見出しを持つ**ことを、3社の実物貸借対照表で確証：
  - あと会、広島常光福祉会、三篠会いずれも
    `固定資産 ＞ （基本財産）/（その他の固定資産）＞ 個別科目（土地・建物…）` の3段。
  - 流動資産・流動負債・固定負債は中間見出しなしで科目が直接並ぶ。
  - `次期繰越活動増減差額`配下に`（うち当期活動増減差額）`の注記行（金額付き）が存在。
- **読み替えルール（CSV列ラベルではなく別表の実階層に従う）**：
  - 固定資産の `（基本財産）/（その他の固定資産）`（元CSVの内訳グループ列）→ **L3 中区分**
  - その配下のCSV「中区分」列の値（土地・建物等）→ **L4 小区分**
  - 他の大区分は中間見出しなし（CSV中区分列＝L3 のまま）。
- この読み替えにより、階層種別とコード深度の食い違い**31件→0件**に解消。
- ※三篠会で`その他の積立金`配下に法人固有の積立金（`川崎市障害児者…`等）が出現。
  これらは法人別 local で吸収する（正典には`その他の積立金`を見出しとして持つのみ）。

### 2-5. 「金額あり/見出し」フラグ：階層から推定→照合で補正（確定）
- `is_amount`：子を持たない末端行＝1（金額あり）、子を持つ中間見出し＝0。
- 結果：金額あり 658行 / 見出し 2行（下記2-6の階層修正後の値）。
- **名前ベースの is_total 判定は廃止**（`室料差額収入`の「差額」誤検出、本物の合計行漏れ）。
  集計行（計・合計・差額・残高）の確定は、後段の照合フェーズで実PDFの集計行と
  突き合わせて行う（合意方針「推定→照合で補正」に沿う）。

### 2-6. 元CSVの階層付けミスを原本で確認・修正（確定）
元CSVに、`事業負担金収入（公費）`が大区分直下の独立した中区分として立ち、その下に
`事業負担金収入（一般）`が小区分でぶら下がる誤りがあった（CF・PLの2箇所）。
**社会福祉法人会計基準の原本で確認**し、正しい階層に修正：
- 正：大区分`介護保険事業収入` → 中区分`介護予防・日常生活支援総合事業収入`
  → 小区分`事業費収入` / `事業負担金収入（公費）` / `事業負担金収入（一般）`（3つ並列）
- CF（行19-21）/ PL（行297-299）を同型に修正。元CSVは保持せず書き換えた。
- 結果：誤った独立中区分が解消し、見出し行 4→2、金額あり行 656→658。総行数660は不変。
- ※「（公費）」を含む他の中区分（外来診療収入等）は配下に「（一般）」を持たず、
  別表上も正しく中区分。今回の修正対象外。

## 3. コード体系（確定仕様）

```
形式: {計算書}-{L0}-{L1}-{L2}-{L3}-{L3.5}-{L4}
  計算書 = CF / PL / BS（英字）
  各セグメント = 2桁ゼロ埋め数字
  00 = 該当レベルなし（実在する区分は必ず01以上）
  連番 = 別表の出現順、親パススコープ内で採番
```

**L0 対応表**（用語集として保存）：

| L0 | 区分名 |
|---|---|
| 01 | 収入（CF） |
| 02 | 支出（CF） |
| 03 | 収益（PL） |
| 04 | 費用（PL） |
| 05 | 繰越・基本金振替等（PL テーブル4） |
| 06 | 資産（BS） |
| 07 | 負債（BS） |
| 08 | 純資産（BS） |

**L1 対応表**：

| 計算書 | L1=01 | L1=02 | L1=03 |
|---|---|---|---|
| CF | 事業活動 | 施設整備等 | その他の活動 |
| PL | サービス活動増減 | サービス活動外増減 | 特別増減 |
| BS | （L1なし＝00） | | |

**採番例**：
```
CF-01-01-01-02-01-01  介護保険事業収入>居宅介護料収入>（介護報酬収入）>介護報酬収入
BS-06-00-02-01-00-01  資産>固定資産>（基本財産）>土地
BS-06-00-02-02-00-02  資産>固定資産>（その他の固定資産）>建物
```
→ `基本財産`配下の土地と`その他の固定資産`配下の建物が、親が違うため別コードになる。
  同名衝突（最大9箇所）を構造的に防止＝前回の主キー重複が発生しない。

## 4. 成果物の列構成（確定・最新形式）

`seiten_master.csv` は、各論理レベルの**コードセグメント列**を、対応する**名前列の直前**に
並べた19列構成。これにより「区分名 → そのレベルのコード」を列単位で直接引ける
（照合パーサがレベル別に突合できる形）。

| # | 列名 | 内容 |
|---|---|---|
| 1 | `code` | 完全コード `{計算書}-{L0}-{L1}-{L2}-{L3}-{L3.5}-{L4}` |
| 2 | `計算書` | CF / PL / BS |
| 3 | `別表` | 別表第一/第二/第三（出典名） |
| 4 | `テーブル` | 元CSVのテーブル番号 |
| 5 | `L0` | L0コード（01〜08, 2桁） |
| 6 | `区分` | L0の名前（収入/支出/収益/費用/繰越/資産/負債/純資産） |
| 7 | `L1` | L1コード（2桁, BS等は00） |
| 8 | `活動・部` | L1の名前（事業活動 等。無い場合は空） |
| 9 | `L2` | L2コード（2桁） |
| 10 | `大区分` | L2の名前 |
| 11 | `L3` | L3コード（2桁） |
| 12 | `中区分` | L3の名前 |
| 13 | `L3.5` | L3.5（内訳グループ）コード（2桁, 無ければ00） |
| 14 | `内訳グループ` | L3.5の名前 |
| 15 | `L4` | L4コード（2桁, 無ければ00） |
| 16 | `小区分` | L4の名前 |
| 17 | `階層種別` | この行の最深レベル（大区分/中区分/内訳グループ/小区分） |
| 18 | `正規名` | この行の最深レベルの科目名（突合キーの元） |
| 19 | `is_amount` | 金額あり=1 / 見出し=0 |

**行イメージ**（CF 介護報酬収入）：
```
code=CF-01-01-01-02-01-01
L0=01 区分=収入 / L1=01 活動・部=事業活動 / L2=01 大区分=介護保険事業収入 /
L3=02 中区分=居宅介護料収入 / L3.5=01 内訳グループ=（介護報酬収入） /
L4=01 小区分=介護報酬収入 / 階層種別=小区分 / 正規名=介護報酬収入 / is_amount=1
```

1. **集計行（is_total）の確定**：名前ベース判定は廃止済み。実PDFの計・合計・差額・残高

# 引き継ぎプロンプト — 社福決算DB マスタ拡充タスク（正典ベース）

このメッセージを最初に読んでください。あなたには、社会福祉法人 決算データベース化
プロジェクトの**勘定科目マスタ拡充**という、独立した一タスクをお願いします。
別チャットで進行中の本体作業から切り出したサブタスクです。**結果（追加タプル列と
対応表）だけを生成**してください。実際のコード反映と検証は依頼元チャットが行います。

## 添付するもの（このプロンプトと一緒に渡されます）
- `マスタ拡充_フォーマット定義書.md` … コード採番ルールと記述フォーマット。**最重要・厳守**。
- `masters.py` … 拡充対象の現行マスタ（既存の全コード体系がここにある）。
- `master_candidates.csv` … 440社の実取り込みで出た「マスタ未収載の大区分・小科目」候補。
  法人数（出現社数）付き。これが「実態の正解」に近い証拠データ。
- 社会福祉法人会計基準の別表（正典）… 下記URLから取得する。

## 正典（社会福祉法人会計基準）の所在
勘定科目は厚労省令の別表で固定されている（クローズドな集合）。以下を参照:
- 別表第一=資金収支計算書勘定科目、別表第二=事業活動計算書勘定科目、
  別表第三=貸借対照表勘定科目。
- 取得元の例（千葉県の手引きにPDF全文あり）:
  https://www.pref.chiba.lg.jp/kenshidou/shafuku-hojin/documents/01_tebikip71-p157.pdf
- 一次資料（省令本体）:
  https://www.mhlw.go.jp/stf/newpage_13281.html （関連通知・別表へのリンク）
- ※URLが取得できない場合は web_search で「社会福祉法人会計基準 別表第一 資金収支計算書
  勘定科目」等を検索し、大区分の正規名一覧を正典として確定すること。

## プロジェクトの背景（なぜこの作業が必要か）
- ゴールは全国 約5万法人（当面 手元440社）の社福決算をDB化し縦横分析できるようにすること。
- 現行の parser/engine と masters.py は **わずか2法人（神奈川厚生福祉会・あしたか太陽の丘）
  だけで構築**された。この2社は介護保険・医療・児童・老人福祉系の事業を持たなかったため、
  **それら事業の収入大区分が masters.py から丸ごと欠落**している。
- 結果、440社取り込みで「大区分が resolve_global できず parent 空のまま採番され、
  拠点をまたいで1コードに潰れ、fact主キー重複（FK/PK問題）が大量発生」した。
  さらに held（取込不可）も多発。**根因はマスタの大区分不足**と特定済み。

## あなたのゴール（成果物）
`マスタ拡充_フォーマット定義書.md` の「4. 作るもの」に従い、以下を生成する:
1. CF_MASTER / PL_MASTER / BS_MASTER の**末尾に追記する追加タプル列**
   （Pythonリテラル。各行に `# 法人数NNN` 等の根拠コメント付き）。
2. 追加した大区分の **正規名 ↔ 採番コード 対応表**。
3. 別表にあるが440社に出なかった大区分の扱い（網羅 vs 実出現）の論点整理。

## 進め方（推奨手順）
1. `マスタ拡充_フォーマット定義書.md` を熟読し、コード体系（CF-{区分}-{INC/EXP}-{連番} 等）と
   「幹=大区分のみ」「既存は変更せず末尾追記」「正規名はnormalizeで突合」を頭に入れる。
2. `masters.py` の現行 CF_MASTER/PL_MASTER/BS_MASTER の **全code集合**と**全正規名集合**を把握。
3. `master_candidates.csv` の「大区分候補（parent空）」を法人数降順で確認。これが追加の主対象。
4. 正典（別表）と突合し、各候補が**別表に実在する正規大区分か**を確認しながら採番。
   - 別表にある → 追加。コードは定義書のルールで既存の続き番号。
   - 別表になく法人独自の造語/表記ゆれ → 追加しない（local側で吸収されるべきもの）。
5. 自己チェック（定義書「5. 検証観点」: コード重複0 / 正規名のnormalize衝突なし /
   大区分のみ / タプル要素数・型一致）。
6. 成果物3点を出力。**masters.py 全体を書き換えて出すのではなく、追加分のタプル列**を出す。

## 重要な制約（必ず守る）
- **大区分のみ**を足す（CF/PL）。BSは中区分まで。小区分・明細は入れない。
- **既存行を変更・削除しない。末尾追記のみ**（採番順を変えると依頼元の神奈川基準線
  sha256 が壊れる）。
- 別表にある正規科目だけ。造語・表記ゆれは入れない。
- 迷ったら「実態（440社集計の法人数）」と「正典（別表）」の両方が支持するものを優先。
- これは引き継ぎ元の設計思想（no overfitting・証拠ベース）に沿った作業である。
  1社や少数社だけに出る曖昧な科目を急いで足さない。

## 最初にやること
1. 3つの添付（定義書 / masters.py / master_candidates.csv）を読み込む。
2. 正典（別表第一・第二・第三）を取得 or 検索で確定する。
3. master_candidates.csv の大区分候補（法人数が多い順）から、別表と突合して追加タプルを作る。
4. 成果物3点を出力する。

完成した追加タプル列と対応表を、依頼元チャットに貼り戻してください。
依頼元が masters.py へ反映し、神奈川基準線（fact 3,326 / sha256 / NG=0）を確認後、
440社を再取り込みして FK/PK と held の減少を検証します。
